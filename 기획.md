## "Another Day" 앱의 핵심은 '유연함'과 '정직한 수치'이다.

## "패스"라는 개념 도입 이유는 사용자가 죄책감 없이 지속 가능한 습관을 형성하게 하려는 의도이다.

# 1. 기본 데이터 설계

- 루틴은 "매일" / "주N회" 로 설정한다.
- 루틴 횟수 (Target): N 회
- 인증 횟수 (Done): 실제 사진 인증한 횟수
- 패스 한도 (Pass Limit): 7 - N 회
- 현재 사용한 패스 (Pass Used): 사용자가 '패스' 버튼을 누른 횟수

# 오늘 (Today) 탭 페이지

- 오늘의 달성률 : **"내가 오늘 하기로 한 일을 얼마나 해냈는가"**가 중요하다.
- ** 계산 방식 ** : 인증 완료 수 / 전체 목표 수 - 패스 수
- 분모에서 패스한 항목은 뺀다
- 이유: 패스를 눌렀다는 것은 "오늘은 이 루틴을 하지 않기로 계획을 수정했다"는 뜻이다. 하지 않기로 한 일을 하지 않았다고 해서 달성률이 깎이면 사용자는 '패스' 기능을 쓸 이유가 없어진다.

# 내목표 탭 페이지

## 오늘 완료 2/3 카드

- 이 카드의 목적이 **"오늘의 요약"**이라면 오늘 탭과 로직이 같아야 합니다.
- 만약 3개 중 2개 인증, 1개 패스라면 "2/2 완료 (1 패스)" 와 같은 방식으로 표현하는 것이 훨씬 직관적이다.
- 이유: "2/3"이라고 적혀 있으면 사용자는 본능적으로 "나머지 1개는 실패했나?"라고 느끼게 됩니다. 패스는 '제외'된 것이지 '실패'한 것이 아니므로, 분모를 2로 줄여서 보여주거나 패스 숫자를 별도로 표기해 주는 것이 정직한 수치 설계에 부합하다.

# 통계 페이지

## 주간 및 월간 통계 (Weekly/Monthly)

- 조금 더 정교한 설계가 필요하다. 주 N회 루틴의 특성상 **"전체 기간"**에 대한 달성률을 산출해야 하기 때문이다.

## 주간 통계 계산 로직

### ① 요일별 달성률 (Daily Breakout)

- 각 요일의 상태에 따라 0% 또는 **100%**로 이분법적으로 계산합니다.
  - 100% (성공)인 경우:
    - 인증 완료: 사진 인증을 한 날.
    - 정상 패스: 패스 한도(7-N) 내에서 '패스' 버튼을 누른 날.
  - 0% (미달)인 경우:
    - 방치: 인증도 패스도 하지 않고 하루가 지난 날.
    - 한도 초과: (패스 버튼을 비활성화하지 않았을 경우) 한도를 초과하여 패스한 날은 '미달'로 처리.

### ② 주간 총합 달성률 (Weekly Total)

- 단순 요일별 평균이 아니라, 주간 전체 목표 이행률을 보여줘야 한다.
  - 매일 루틴: (실제 완료 횟수 / 7일) \* 100
  - 주 N회 루틴: (실제 완료 횟수 / 설정한 N회) \* 100
  - 특이점:
    - 인증을 $N$번 다 하면 패스 여부와 상관없이 **100%**가 됩니다.
    - 패스는 이 공식의 수치(%)에는 영향을 주지 않지만, '미달'을 방지하여 요일별 통계를 깨끗하게 유지해주는 방어막 역할을 합니다.

### ③ 루틴별 현황 (Per-Routine Stats)

- 각 습관이 한 주 동안 얼마나 잘 지켜졌는지 개별적으로 보여줍니다.
  - 표기 예시:
    - [매일 루틴]
      - 표기: 완료 5 / 미달 2
      - 이유: 매일 루틴은 패스가 없으므로 이분법적으로 보여주는 것이 명확합니다.
    - [주 3회 루틴]
      - 표기: 완료 2/ 미달 1 (패스 1)
      -
  - 계산:
    - 완료: 인증 횟수
    - 패스: 사용한 패스권 횟수
    - 미달: (인증도 패스도 안 한 날) + (패스 한도 초과분)

- 시나리오 예시 (주 3회 목표, 패스 한도 4회)
  요일,상태,요일 달성률,패스 잔여량,주간 총합 달성률
  월,패스,100%,3장,0% (인증 0/3)
  화,인증,100%,3장,33% (인증 1/3)
  수,패스,100%,2장,33% (인증 1/3)
  목,방치,0%,2장,33% (인증 1/3)
  금,인증,100%,2장,66% (인증 2/3)
  토,패스,100%,1장,66% (인증 2/3)
  일,패스,100%,0장,66% (인증 2/3)
  - 결과 요약:
    - 요일별: 목요일 제외 모두 100%. (차트상으로는 목요일만 푹 꺼짐)
    - 주간 총합: 66%. (목표 3회 중 2회만 인증했으므로)
    - 사용자 피드백: "이번 주에 패스를 다 써버렸네요! 목요일에 방치한 것 때문에 목표 달성에 실패했어요."

### 월간 통계

- 월간 통계는 각 주차의 달성률을 평균 내거나, 한 달 전체 목표 횟수(예: 주 3회 x 4주 = 12회) 대비 실제 인증 횟수로 계산한다.
- 여기서도 패스는 "실패가 아님"을 증명하는 수단일 뿐, 달성률을 높여주는 '성공' 데이터는 아닙니다.

# 사용자 입장에서의 UX 흐름

- **"목표를 다 채웠어도 패스를 눌러야 100%다"**라는 규칙
- 툴팁 제공: 0%인 요일을 클릭하면 "목표는 달성했지만, 오늘 기록이 없어요. 패스를 눌러 기록을 완성하세요!"라는 문구를 보여주세요.
- 색상 차별화:
  - 목표 달성 전의 미달: 빨간색 (경고)
  - 목표 달성 후의 미달: 회색 (단순 기록 누락)
